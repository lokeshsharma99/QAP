🤖 What is Quality Autopilot?

⚖️ The Agentic Judge

The invisible quality gate. A post_hook on every workflow step — never a team member.

🚨
NEVER bypass the Judge. NEVER bypass HumanReviewGateStep. If Judge confidence < 90%, the Human Lead MUST review. No exceptions.

🎯 Trust Logic

confidence ≥ 90%
✅ AUTO-APPROVE — artifact proceeds to next pipeline step
confidence < 90%
⏸ HumanReviewGateStep — artifact held for Human Lead in AgentUI
confidence < 50%
❌ AUTO-REJECT — sent back to producing agent with specific feedback

📋 Judge Variants (DoD Checklists)

VariantReviewsKey Checks
Gherkin Judge Scribe output BA-readable? Reusable steps? Syntax valid? Traceability to Jira AC?
Code Judge Engineer output No hardcoded sleep? Modular POM? eslint passes? Type-check passes? Look-Before-You-Leap followed?
Data Judge Data Agent output PII masked? Unique constraints satisfied? No production data? Cleanup queries present?
Healing Judge Medic output Verification passes 3×? logic_changed == False? Diff is surgical (single locator line)?

📄 JudgeVerdict Contract

class JudgeVerdict(BaseModel):
    artifact_type: str           # "gherkin" | "code" | "data" | "healing"
    agent_id: str               # ID of the producing agent
    confidence: float           # 0.0 – 1.0
    passed: bool                # True if confidence >= 0.90
    checklist_results: dict[str, bool]  # each DoD item pass/fail
    rejection_reasons: list[str]      # empty if passed
    requires_human: bool        # True if confidence < 0.90

🩺 Medic Constraints (Non-Negotiable)

⛔
Medic is ONLY allowed to change locator selectors and wait strategies.
NEVER: assertions, test flow, business logic, method names, imports.
Edits must be surgical — one locator line, not a file rewrite.
Must produce a unified diff for human review.
Verification run must pass 3× consecutively before PR.

📦 How Knowledge Bases are Created

# db/session.py
def create_knowledge(name: str, table_name: str) -> Knowledge:
    return Knowledge(
        name=name,
        vector_db=PgVector(
            db_url=db_url,
            table_name=table_name,
            search_type=SearchType.hybrid,
            embedder=OllamaEmbedder(id="qwen3-embedding:4b", dimensions=2560),
        ),
    )
ℹ️
Git-sync hook: A .git/hooks/post-commit script triggers the Librarian to re-index codebase_vectors on every commit to main/develop. This keeps the Engineer's KB always current.

🚫 Anti-Patterns

Things that will break the system — and what to do instead.

❌ Never Do This✅ Do This Instead
Use LangChain / CrewAI / AutoGen Use agno.agent.Agent, agno.team.Team, agno.workflow.Workflow only
Hardcode time.sleep() or waitForTimeout() Use Playwright's built-in auto-waiting
Put all agents in one file One directory per agent with agent.py, instructions.py, __init__.py
Skip Pydantic contracts between agents Every hand-off uses a typed contract from contracts/
Use CSS selectors or XPath in tests Use data-testid, role, or text strategies
Medic changes business logic or assertions Medic changes ONLY locator selectors and wait strategies
Use SQLite for agent persistence PostgreSQL + PgVector via db/session.py
Engineer writes code before checking Site Manifesto Follow Look-Before-You-Leap: Manifesto → KB → MCP → write
Bypass the Judge or HumanReviewGateStep Every artifact passes JudgeVerdict. If <90%, hold for Human Lead.
Hardcode secrets or API keys in code Use environment variables via getenv()
Work on Phase N+1 before Phase N gate passes Respect the gated roadmap. Check CHECKLIST.md first.
Nest code more than 3 levels deep Flatten with early returns and helper functions
docker compose up -d qap-ui after a build docker compose up -d --force-recreate qap-ui

🔐 Security Rules

🔒
NEVER output .env contents, API keys, tokens, passwords, database credentials, or connection strings.
Do not include example formats, redacted versions, or placeholder templates for secrets.
All Playwright test containers run with network_mode: "none" (sandbox).
All user input is validated at system boundaries via Pydantic contracts.
No SQL string concatenation — use parameterised queries only.
